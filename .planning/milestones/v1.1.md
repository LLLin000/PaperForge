## Key Decisions

1. **Config precedence:** explicit overrides > env > JSON nested > JSON top-level > defaults
2. **Canonical PaddleOCR env var:** `PADDLEOCR_API_TOKEN`
3. **CLI exit codes:** int return for testability; worker functions imported at module level for patchability
4. **Python 3.14 workaround:** `importlib.util` with `sys.modules` pre-registration for dataclass compatibility
5. **Deterministic fixtures:** generated once and committed; generation script available but not run in CI
6. **Rollback pattern:** track written files, save original state, clean up on exception (not full filesystem snapshot)
7. **Repair idempotency:** dry-run by default; `--fix` required for actual repairs

## Known Limitations

1. **BBT bare path normalization:** Not auto-converted in `load_export_rows()`. Requires BBT to emit `storage:` prefix or manual path configuration.
2. **Repair scan scaling:** O(n*m) per-record rglob in large vaults. Acceptable for current use cases.
3. **Pre-existing test failures:** `test_base_preservation.py` and `test_base_views.py` have `ModuleNotFoundError: No module named 'pipeline'` — import path issues from Phase 3, unrelated to v1.1 scope.
