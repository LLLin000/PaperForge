### Task 2: Create OCR-Complete Fixture (DEEP-06)

**Files:**
- New: `tests/sandbox/ocr-complete/TSTONE001/fulltext.md`

- New: `tests/sandbox/ocr-complete/TSTONE001/figure-map.json`
- New: `tests/sandbox/ocr-complete/TSTONE001/chart-type-map.json`

**Fixture content:**
- `fulltext.md`: Synthetic markdown with realistic figure captions (e.g., "Figure 1: Biomechanical comparison of suture anchor fixations...", "Table 1: Patient demographics...") and page markers (`<!-- page 1 -->`, `<!-- page 2 -->`).
- `meta.json`: `{ "zotero_key": "TSTONE001", "ocr_status": "done", "page_count": 3, "generated_at": "..." }`
- `figure-map.json`: Generated by running `python skills/literature-qa/scripts/ld_deep.py figure-map tests/sandbox/ocr-complete/TSTONE001/fulltext.md --key TSTONE001 --out tests/sandbox/ocr-complete/TSTONE001/figure-map.json`
- `chart-type-map.json`: Generated by running `python skills/literature-qa/scripts/ld_deep.py chart-type-scan tests/sandbox/ocr-complete/TSTONE001/figure-map.json --out tests/sandbox/ocr-complete/TSTONE001/chart-type-map.json`

**Note:** The fixture must be committed to git. Generate once, commit, and never regenerate (to keep tests deterministic).

**2a. Write fixture generation script (for one-time use):**
- Create `tests/sandbox/generate_ocr_fixture.py` that:
  1. Reads `tests/sandbox/exports/骨科.json` to get TSTONE001 metadata
  2. Writes synthetic `fulltext.md` with figure/table captions
  3. Runs `ld_deep.py figure-map` and `chart-type-scan` to produce JSON
  4. Writes `meta.json` with `ocr_status: done`

**2b. Generate and commit fixtures:**
- Run `python tests/sandbox/generate_ocr_fixture.py`
- Verify files are created
- Add `tests/sandbox/ocr-complete/` to `.gitignore`? No — commit fixtures (deterministic)
- But add `tests/sandbox/00_TestVault/` to `.gitignore` (generated vault output)
