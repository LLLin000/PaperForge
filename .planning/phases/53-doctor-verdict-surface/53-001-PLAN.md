  <action>
    Add the following new helper functions to `paperforge/worker/status.py` (before `run_doctor()`), then wire them into `run_doctor()` replacing the current `sys.version_info` check:

    === 1. _read_plugin_data(vault: Path) -> dict ===
    Read `.obsidian/plugins/paperforge/data.json` relative to vault.
    - Return empty dict if file not found, invalid JSON, or not a dict.
    - Do NOT raise exceptions on missing file.

    === 2. _resolve_plugin_interpreter(vault: Path, plugin_data: dict) -> tuple[str, str, list[str]] ===
    Replicate the plugin's `resolvePythonExecutable()` logic in pure Python.
    Returns (interpreter_path: str, source: str, extra_args: list[str]).

    Detection order (per Phase 51 decision D01):
    1. **Manual override**: If `plugin_data` has `python_path` (non-empty, trimmed) AND the path exists on disk via `os.path.exists()`, return it with source="manual".
    2. **Venv candidates** (Windows `Scripts/python.exe` pattern, use `os.path.exists`):
       - `vault /.paperforge-test-venv/Scripts/python.exe`
       - `vault /.venv/Scripts/python.exe`
       - `vault /venv/Scripts/python.exe`
       Return first found with source="auto-detected" and empty extra_args.
    3. **System candidates** — use `subprocess.run()` to test each:
       - `["py", "-3", "--version"]`
       - `["python", "--version"]`
       - `["python3", "--version"]`
       For each, run with `timeout=5`, `capture_output=True`, check returncode==0 and output contains "Python".
       Return first successful with source="auto-detected" and the candidate's extra_args.
    4. **Fallback**: Return `("python", "auto-detected", [])`.

    Notes:
    - Use `os.name == "nt"` — the Scripts/python.exe pattern is Windows-specific.
    - On non-Windows, venv candidates use `bin/python` (e.g., `<venv>/bin/python`).
    - Use `import subprocess` at module top level (add to existing imports).
    - Handle permissions errors gracefully — treat candidate as not found.

    === 3. _query_resolved_version(interp: str, extra_args: list[str]) -> tuple[str | None, tuple[int, int, int] | None] ===
    Run `[interp] + extra_args + ["--version"]` via subprocess.
    Parse output like "Python 3.11.5" into a version tuple.
    Return (version_string, (major, minor, micro)) or (None, None) on failure.

    === 4. _query_resolved_package(interp: str, extra_args: list[str], package_name: str) -> dict | None ===
    Run `[interp] + extra_args + ["-m", "pip", "show", package_name]` via subprocess.
    Parse pip show output (Name:, Version:, Location:, etc.) into a dict.
    Return None if pip show fails (non-zero exit, stderr, or package not found).
    Key fields to extract: "Name", "Version", "Location", "Editable project location".
    Use `timeout=15` (pip show can be slow).

    === Wire into run_doctor() ===

    At the top of `run_doctor()`, BEFORE the current `sys_version = sys.version_info` block:

    1. Call `plugin_data = _read_plugin_data(vault)`
    2. Call `interp, source, extra_args = _resolve_plugin_interpreter(vault, plugin_data)`
    3. Call `_query_resolved_version(interp, extra_args)` to get resolved_interpreter_version and version_tuple
    4. Call `_query_resolved_package(interp, extra_args, "paperforge")` to get pf_pkg_info

    ADD the following new checks AFTER the existing `add_check("Python 环境", ...)` block for sys.version:

    **Check A — Resolved Interpreter Path (DOCTOR-01):**
    ```

        f"已解析解释器: {interp} (来源: {source})")
    ```

    **Check B — Resolved Interpreter Version (DOCTOR-01):**
    If version_tuple is not None and >= (3, 10, 0):
      add_check("Python 环境 (插件)", "pass",
          f"Python {version_str}")
    elif version_tuple is not None:
      add_check("Python 环境 (插件)", "fail",
          f"Python {version_str} (需要 3.10+)",
          f"升级解释器 {interp} 到 Python 3.10 或更高版本")
    else:
      add_check("Python 环境 (插件)", "fail",
          f"无法获取 {interp} 的 Python 版本",
          f"验证 {interp} 是一个有效的 Python 解释器")

    **Check C — Package Installed + Version + Drift (DOCTOR-02):**
    If pf_pkg_info is not None:
      pkg_version = pf_pkg_info.get("Version", "?")
      pkg_location = pf_pkg_info.get("Location", "?")
      expected_version = __import__("paperforge").__version__
      if pkg_version == expected_version:
        add_check("PaperForge 包", "pass",
            f"v{pkg_version} 已安装 → {pkg_location}")
      else:
        add_check("PaperForge 包", "warn",
            f"v{pkg_version} 已安装 (插件版本 v{expected_version}) — 版本不匹配",
            f"运行: {interp} -m pip install --upgrade git+https://github.com/LLLin000/PaperForge.git")
    else:
      add_check("PaperForge 包", "fail",
          f"PaperForge 未安装在 {interp} 中",
          f"运行: {interp} -m pip install -e .")

    **Check D — Wrong-Environment Detection (DOCTOR-02):**
    Compare the resolved interpreter's paperforge package path with the current doctor process's paperforge location.
    - Get current_package_path from `os.path.dirname(__import__("paperforge").__file__)`
    - If pf_pkg_info has "Location": compare:
      If Location is a string AND current_package_path exists:
        If Location != current_package_path:
          add_check("PaperForge 包", "warn",
              f"包路径不一致: 已解析解释器 → {pkg_location} | 当前诊断进程 → {current_package_path}",
              "建议: 使用已解析解释器运行 doctor，或统一 Python 环境")

    **Keep the existing `sys.version_info` check** (it reports the doctor's own Python version, useful for comparison), but move it to appear under "Python 环境" category BEFORE the new plugin-resolved checks. This way the user sees both:
    1. "Python 环境 — Python 3.11.5 (当前诊断进程)"
    2. "Python 环境 (插件) — ..." (the resolved interpreter)

    **Do NOT remove existing checks.** Only add new ones.

    === Pass vault to run_doctor() ===
    `run_doctor(vault)` already accepts `vault: Path` as first argument (line 197). The CLI at cli.py:451 calls `run_doctor(vault)`. This is already correct — no change needed in cli.py.

    - Do NOT break backward compat for the `sys.version_info` check — keep it.

       e. If import failed: add_check with "fail" status, message includes label, fix includes the per-package pip command from the manifest.

    Example output:
    ```
    [PASS] Python 环境 — requests 已安装 (2.31.0)
    [PASS] Python 环境 — Pillow 已安装 (10.2.0)
    [FAIL] Python 环境 — PyYAML 缺失
          修复: pip install pyyaml>=6.0
    ```

    === Part B: Final verdict (DOCTOR-04) ===

       ```python
       has_warn = any(status == "warn" for _, status, _, _ in checks)
       ```

    2. Determine verdict:
       - If has_fail: verdict = "FAIL", verdict_color = "red"
       - If has_warn: verdict = "WARN", verdict_color = "yellow"
       - Else: verdict = "OK", verdict_color = "green"

    3. Determine recommended next action:
       Build a terse recommendation based on which categories have failures:
       ```python
       fail_categories = set()
       for cat, status, _, fix in checks:
           if status == "fail" and fix:
               fail_categories.add(cat)
       ```
       - If "PaperForge 包" in fail_categories and no other fails: "Recommended: run pip install via the resolved interpreter"
       - If "Python 环境" or "Python 环境 (插件)" in fails: "Recommended: install/upgrade Python to 3.10+"
       - If "Vault 结构" or config-related fails: "Recommended: run `paperforge sync`"
       - If "OCR 配置" fails: "Recommended: configure PADDLEOCR_API_TOKEN in .env"

       - If no fails but has warns: "Recommended: `paperforge doctor` indicates warnings — review above"
       - If all pass: "Recommended: `paperforge sync` to ensure index is current"
       - Default fallback: "Recommended: review the failing checks above"

    4. Print the verdict block:
       ```python
       separator = "=" * 40
       print(f"\n{separator}")
       
       # Color-code: use ANSI escape codes
       GREEN = "\033[92m"
       YELLOW = "\033[93m"
       RED = "\033[91m"
       RESET = "\033[0m"
       
       if verdict == "OK":
           tag = f"{GREEN}[OK]{RESET}"
       ```

    5. Disable color codes when not writing to a TTY (pipe/file):
